### Antifragile
-----
```
※ 확장성, 안정성 강화 , 지속적인 배포 / DevOps와 Cloud Native가 생겨남

- Auto Scaling(자동 확장성) : 인스턴스를 자동으로 증가할 수 있는 환경
- Microservices : - 넷플릭스처럼 복잡한(나눈) 서비스 , 넷플릭스가 거대한 클라우드 환경을 사용중이고 
                  - Spring Cloud가 이를 기부받아 사용중이며 아직까지 넷플릭스의 기능을 사용하는 것도 있음. 
                  - 전체 서비스를 구성하는 전체 모듈이나 기능을 독립적으로 개발 & 배포하는 세분화된 서비스
                  - 클라우드 네이티브 환경에서 적개는 수십개 많게는 수백개의 도메인이 분리되어 개발되는 경우
- Chaos engineering : - 시스템이 급격하고 예측하지 못한 환경이라도 견딜 수 있고 신뢰성을 쌓기위한 시스템의 방법과 규칙
- CI / CD : - 지속적인 통합 , 지속적인 배포
            - 수십 수백개의 서비스를 빌드 & 배포하는 작업
            - 클라우드 네이티브 환경에서 더 활용되고 빠질 수 없는 기능
```
